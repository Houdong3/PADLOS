# PADLOS

This repository releases the inference pipeline for the PADLOS PGM module and the downstream ACM instance extraction module.

The root entrypoints are:

- [run_pgm.py](/media/user/drive11/padlos/PADLOS/run_pgm.py)
- [run_acm.py](/media/user/drive11/padlos/PADLOS/run_acm.py)

`run_pgm.py` can generate:

- binary masks for the PGM segmentation output
- depth maps for the downstream ACM module

`run_acm.py` uses:

- the original input image
- PGM binary masks
- PGM depth maps

to generate instance extraction results.

## Requirements

Install the PGM runtime dependencies:

```bash
pip install -r PDSegmentor/requirements.txt
```

Install the ACM runtime dependencies:

```bash
pip install -r instance_extractor/requirements.txt
```

## Usage

Recommended workflow:

1. run PGM to generate masks and depth maps
2. run ACM to generate instance extraction results

Run both mask and depth inference:

```bash
python run_pgm.py --img-path data/imgs --device auto
```

Run mask inference only:

```bash
python run_pgm.py --img-path data/imgs --task mask --device auto
```

Run depth inference only:

```bash
python run_pgm.py --img-path data/imgs --task depth --device auto
```

Run on CPU explicitly:

```bash
python run_pgm.py --img-path data/imgs --device cpu
```

Save raw depth arrays in addition to PNG files:

```bash
python run_pgm.py --img-path data/imgs --task depth --save-npy
```

Run ACM instance extraction using the outputs generated by PGM:

```bash
python run_acm.py --img-path data/imgs
```

Run ACM on a single image:

```bash
python run_acm.py --img-path data/imgs/test.png
```

Minimal console output examples:

`run_pgm.py`

```text
Input images: 1 from data/imgs
Saving masks to: /path/to/PADLOS/outputs/masks
Saving depth outputs to: /path/to/PADLOS/outputs/depth
```

`run_acm.py`

```text
Input images: 1 from data/imgs
Using masks from: /path/to/PADLOS/outputs/masks
Using depth from: /path/to/PADLOS/outputs/depth
Saving instances to: /path/to/PADLOS/outputs/instances
```

## Inputs

`--img-path` accepts:

- a single image path
- a directory containing images

Supported formats include `jpg`, `jpeg`, `png`, `bmp`, `tif`, and `tiff`.

## Outputs

Inference results are saved in the root `outputs/` directory:

- [outputs/masks](/media/user/drive11/padlos/PADLOS/outputs/masks): binary segmentation masks named `*_mask.png`
- [outputs/depth](/media/user/drive11/padlos/PADLOS/outputs/depth): depth outputs named `*_depth_gray.png`, `*_depth_color.png`, and optionally `*_depth.npy`
- [outputs/instances](/media/user/drive11/padlos/PADLOS/outputs/instances): ACM instance extraction outputs named `*_instances.png`

## Main Arguments

Useful options for [run_pgm.py](/media/user/drive11/padlos/PADLOS/run_pgm.py):

- `--task {both,mask,depth}`: choose which outputs to generate
- `--device {auto,cuda,cpu}`: choose the inference device
- `--input-size 518`: set inference resolution
- `--threshold 0.05`: set the mask binarization threshold
- `--limit N`: only process the first `N` images for mask inference
- `--grayscale`: save grayscale depth previews instead of colorized ones
- `--show-depth`: display depth figures in a window

Useful options for [run_acm.py](/media/user/drive11/padlos/PADLOS/run_acm.py):

- `--mask-dir`: choose the directory containing PGM masks
- `--depth-dir`: choose the directory containing PGM depth maps
- `--save-dir`: choose where ACM instance results are written
- `--limit N`: only process the first `N` images

## Notes

- If your GPU is not compatible with the installed PyTorch build, use `--device cpu`.
- `run_pgm.py` and `run_acm.py` are the recommended public entrypoints.
